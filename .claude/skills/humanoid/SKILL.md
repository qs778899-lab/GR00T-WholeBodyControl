# Humanoid Robot 工程规范

## 代码实现原则

- 将参考代码链路进行细致拆分，理清每一个环节的输入输出
- 尽量小改动，尽量不修改原代码库的代码文件
- 尽量复用原代码库已有的函数或类

## 帧时间对齐

涉及参考动作（reference）与实际输出动作的对比分析时：

- 注意每一帧时间的严格对齐
- 如果无法严格对齐，需分析并给出原因

## 出现问题时

先给出具体方案，说明每一个环节具体怎么做。

若无法确定具体原因，可以通过增加日志输出的方式进行详细分析判断：
- 日志保存不能占用几十 GB 的空间
- 需要有及时清理的机制

## 问题排查的可行方案

- 可视化
- Test case
- 拆分参量的数值记录文件
